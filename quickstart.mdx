This guide walks you through cloning the repo, configuring your environment, and making your first API call.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js** 18–22 — [nodejs.org](https://nodejs.org)
- **npm** (bundled with Node.js)
- **PostgreSQL** 14 or later — [postgresql.org](https://www.postgresql.org)

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/cafe-acacias/api.git
    cd api
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and fill in the required values:

    ```bash
    cp .env.example .env
    ```

    Open `.env` and update the values below.

    **Application secrets** — generate a unique value for each:

    ```bash .env
    HOST=0.0.0.0
    PORT=1337

    # Generate with: node -e "console.log(require('crypto').randomBytes(16).toString('base64'))"
    APP_KEYS="yourKey1,yourKey2"
    API_TOKEN_SALT=yourApiTokenSalt
    ADMIN_JWT_SECRET=yourAdminJwtSecret
    TRANSFER_TOKEN_SALT=yourTransferTokenSalt
    JWT_SECRET=yourJwtSecret
    ENCRYPTION_KEY=yourEncryptionKey
    ```

    **Database connection** — defaults assume a local PostgreSQL instance:

    ```bash .env
    DATABASE_CLIENT=postgres
    DATABASE_HOST=localhost
    DATABASE_PORT=5432
    DATABASE_NAME=strapi
    DATABASE_USERNAME=strapi
    DATABASE_PASSWORD=strapi
    DATABASE_SSL=false
    ```

    <Note>
      Alternatively, set `DATABASE_URL` to a full connection string (e.g., `postgres://user:pass@host:5432/dbname`) instead of the individual variables above.
    </Note>

    <Warning>
      Never commit your `.env` file to version control. It is already listed in `.gitignore`.
    </Warning>
  </Step>

  <Step title="Create the database">
    Connect to PostgreSQL and create the database and user if they do not exist:

    ```sql
    CREATE USER strapi WITH PASSWORD 'strapi';
    CREATE DATABASE strapi OWNER strapi;
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run develop
    ```

    Strapi will build the admin panel and start the server. On first run this may take a minute or two.

    You should see output ending with:

    ```
    [2026-03-17 00:00:00.000] info: Strapi started successfully
    ```
  </Step>

  <Step title="Access the admin panel">
    Open [http://localhost:1337/admin](http://localhost:1337/admin) in your browser.

    On first launch, you will be prompted to create an administrator account. Fill in your name, email, and a strong password, then click **Let's start**.

    <Note>
      The admin panel account is separate from API user accounts. See [Authentication](/authentication) for details on API-level registration and login.
    </Note>
  </Step>

  <Step title="Make your first API call">
    With the server running, fetch the list of products:

    ```bash
    curl http://localhost:1337/api/products
    ```

    A successful response returns a paginated JSON payload:

    ```json
    {
      "data": [],
      "meta": {
        "pagination": {
          "page": 1,
          "pageSize": 25,
          "pageCount": 0,
          "total": 0
        }
      }
    }
    ```

    <Note>
      If the endpoint returns a `403 Forbidden`, the **Products** collection's public `find` permission has not been enabled yet. Go to **Settings → Users & Permissions → Roles → Public** in the admin panel and enable the permission.
    </Note>
  </Step>
</Steps>
